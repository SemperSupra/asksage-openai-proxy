# Contributing to this Project

**This repository is open for public visibility and reference, but participation is strictly invite-only.**

Only users who are explicitly added as repository collaborators are permitted to open Issues or Pull Requests.

## Policy

Any Issue or Pull Request opened by a non-collaborator will be automatically:
1. Labeled `unapproved`.
2. Given a short explanatory comment.
3. Closed.

There is no process for requesting access. Please do not ask to be added as a collaborator.

## For Collaborators

### Issues
When opening an issue, please provide full reproduction steps and environment details.

### Pull Requests
- Keep changes small and focused.
- Ensure all tests pass.
- Follow the existing code style.

## Security

If you discover a security vulnerability, please report it via private channels. Do not disclose it publicly in issues or pull requests.
